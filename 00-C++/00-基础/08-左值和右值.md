# C++ 左值与右值

## 1. 基本定义

### 1.1 传统区分（C++11 之前）

- **左值 (lvalue)**：可以取地址、有名字的表达式。通常位于赋值运算符左侧。
- **右值 (rvalue)**：临时值、字面量，不能取地址，通常位于赋值运算符右侧。

```cpp
int x = 42;   // x 是左值，42 是右值
int* p = &x;  // 可以对左值取地址
// int* q = &42;  // 错误：不能对右值取地址
```

### 1.2 C++11 之后的精细分类

C++11 引入移动语义后，将值类别细分为五类：

``` bash
        expression
        /        \
    glvalue     rvalue
    /    \      /    \
lvalue  xvalue prvalue
```

- **lvalue**（左值）：传统左值，有身份且不能被移动。
- **xvalue**（将亡值）：有身份但可以被移动（如 `std::move` 的返回值）。
- **prvalue**（纯右值）：临时对象、字面量，没有身份。
- **glvalue** = lvalue + xvalue（有身份的表达式）
- **rvalue** = xvalue + prvalue（可移动的表达式）

```cpp
int a = 10;
int b = 20;
int&& r = std::move(a);  // std::move(a) 是 xvalue
int c = a + b;           // a+b 是 prvalue
```

## 2. 左值与右值的核心区别

| 特性               | 左值                    | 右值                    |
|-------------------|------------------------|-------------------------|
| 可取地址 (`&`)      | 可以                   | 不可以（除 xvalue 外）   |
| 有名字             | 通常有                 | 通常无（临时对象）       |
| 可修改             | 取决于 const 修饰       | 一般可修改（移动后置空） |
| 生命周期           | 由作用域决定           | 临时对象，可被引用延长   |
| 绑定到引用         | 可绑定到左值引用 `T&`   | 可绑定到右值引用 `T&&`   |

## 3. 变量定义与值类别的关系

### 3.1 变量名永远是左值

**无论变量的类型是什么**（左值引用、右值引用、普通对象），变量名作为表达式时都是左值。

```cpp
int a = 10;           // a 是左值
int& la = a;          // la 是左值引用，但 la 这个名字是左值
int&& ra = 20;        // ra 是右值引用，但 ra 这个名字是左值！

// 验证
void foo(int&)  { cout << "左值引用\n"; }
void foo(int&&) { cout << "右值引用\n"; }

foo(ra);   // 输出 "左值引用" —— 因为 ra 是左值表达式
foo(std::move(ra)); // 输出 "右值引用"
```

**结论**：拥有名字的变量（包括右值引用变量）都是左值表达式。

### 3.2 引用类型影响的是“绑定目标”，而非变量自身的值类别

```cpp
int x = 10;
int& lr = x;     // lr 绑定到左值 x，但 lr 自身是左值
int&& rr = 20;   // rr 绑定到右值 20，但 rr 自身是左值
```

### 3.3 临时对象的生命周期延长

右值引用可以延长临时对象的生命周期，使其与引用变量存活时间一致。

```cpp
struct Big {};
Big create() { return Big(); }

int main() {
    Big&& obj = create();   // 临时 Big 对象的生命周期被延长到 obj 销毁
    // 若无右值引用，临时对象会在表达式结束后析构
}
```

## 4. 为什么 `int&& rri = 42;` 和 `int rri = 42;` 看起来一样？

### 4.1 对于平凡类型（如 int），移动与拷贝无区别

`int` 的拷贝成本极低，移动语义没有优势。所以两种写法最终效果相似。

### 4.2 底层差异仍然存在

| 方面                 | `int rri = 42;`                | `int&& rri = 42;`              |
|---------------------|--------------------------------|--------------------------------|
| 变量类型             | `int`                          | `int&&`                        |
| 存储空间             | 有独立存储                     | 无独立存储，只是引用           |
| 初始化               | 拷贝/直接初始化                | 绑定到临时对象，延长生命周期   |
| `decltype(rri)`     | `int`                          | `int&&`                        |
| 能否重新绑定         | 不适用                         | 不能，引用初始化后不可更改     |
| 作为表达式时的类别   | 左值                           | 左值                           |

### 4.3 代码示例揭示差异

```cpp
#include <iostream>
#include <type_traits>

int main() {
    int a = 42;
    int&& b = 42;

    // 类型不同
    std::cout << std::is_same_v<decltype(a), int> << "\n";      // 1
    std::cout << std::is_same_v<decltype(b), int&&> << "\n";    // 1

    // 修改行为
    a = 100;   // 直接修改对象
    b = 200;   // 修改的是 b 绑定的临时对象

    // 取地址：都可以，但含义不同
    int* pa = &a;   // a 的地址
    int* pb = &b;   // 临时对象的地址（该临时对象的生命周期因 b 而延长）
}
```

## 5. 右值引用的典型用途

### 5.1 移动语义（避免深拷贝）

```cpp
class MyString {
    char* data;
public:
    // 移动构造函数
    MyString(MyString&& other) noexcept 
        : data(other.data) {
        other.data = nullptr;   // 窃取资源，使源对象置空
    }
};
```

### 5.2 完美转发（保持值类别）

```cpp
template<typename T>
void wrapper(T&& arg) {   // 转发引用
    target(std::forward<T>(arg));  // 保持 arg 的左/右值属性
}
```

### 5.3 在容器中高效插入临时对象

```cpp
std::vector<std::string> vec;
vec.push_back("hello");           // 临时 string 是右值 → 移动构造
std::string s = "world";
vec.push_back(s);                 // s 是左值 → 拷贝构造
vec.push_back(std::move(s));      // 转为右值 → 移动构造
```

## 6. 常见误区总结

| 误区                                      | 正解                                      |
|-------------------------------------------|-------------------------------------------|
| `T&&` 变量一定是右值表达式                 | 有名字的 `T&&` 变量是左值表达式           |
| `const T&` 可以绑定一切，不需要 `T&&`      | `T&&` 可以修改右值，且强制移动语义         |
| `int&& x = 10;` 比 `int x = 10;` 高效     | 对内置类型没有区别，对类类型才有           |
| `std::move(x)` 移动了 `x`                 | `std::move` 只是转型为右值引用，真正的移动发生在构造函数/赋值运算符 |

## 7. 快速判断表达式是左值还是右值

- **有名字** → 左值（包括变量、函数名）
- **可取地址** → 左值（除非是 `std::move` 等返回的 xvalue）
- **字面量（字符串字面量除外）** → 右值（字符串字面量是左值，因为其地址不变）
- **临时对象** → 右值
- **`std::move(x)`** → 右值（具体是 xvalue）
- **算术表达式 `a+b`** → 右值（prvalue）

```cpp
"hello"        // 左值（数组左值，可取地址）
42             // 右值
true           // 右值
std::vector<int>{1,2,3}  // 右值（临时对象）
```
