# 系统基础

## 用户与用户组

Linux 靠用户 + 用户组管权限：一个用户只能一个主组，能进很多附加组，进哪个组就有哪个组的权限。

### 用户操作

新增用户 `sudo adduser <用户名>`

删除用户(包括自动创建的用户组和主目录) `sudo userdel -r <用户名>`

只删除用户账号，保留主目录 `sudo userdel <用户名>`

切换用户 `su - <用户名>`
> 切换root用户 sudo su

查看当前用户属于的用户组

```bash
$ groups
jindouchen adm cdrom sudo dip plugdev users lpadmin
```

当前用户属于多个用户组，jindouchen是主用户组

### 用户组

查看当前一共多少用户组 `getent group | cut -d: -f1`

以docker用户组为例：是 /var/run/docker.sock 这个 Unix 套接字文件。Docker 客户端就是通过它和以 root 用户权限运行的后台进程通信的

```bash
$ ls /var/run/docker.sock -l
srw-rw---- 1 root docker 0  4月  7 09:10 /var/run/docker.sock
```

加入docker用户组，这样就不用sudo去启动docker了

指定用户加入docker用户组 `sudo usermod -aG docker <用户名>`
> 重新登录生效

删除指定用户docker用户组 `sudo gpasswd -d <用户名> docker`

验证用户不能使用docker命令 `sudo -u <用户名> docker ps`
> 应该返回 permission denied

查看当前用户属于哪些用户组 `groups`
