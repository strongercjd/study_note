``` bash
tar [选项] [-f 归档文件名] [要打包/解压的文件/目录]
```

`tar` 命令的这些选项/操作均源自英文单词的缩写（部分为传统约定俗成的简写），以下是逐一拆解：  

核心操作(最基础的动作类)

| 选项   | 英文全称           | 中文释义  | 补充说明               |
| ---- | -------------- | ----- | ------------------ |
| `-c` | **c**reate     | 创建    | 核心操作，用于新建归档文件      |
| `-x` | e**x**tract    | 提取/解压 | 从归档文件中释放内容         |
| `-t` | lis**t**（传统简写） | 列出/查看 | 早期Unix工具常将list简写为t |

压缩/解压算法类

| 选项   | 英文全称              | 中文释义         | 补充说明                                  |
| ---- | ----------------- | ------------ | ------------------------------------- |
| `-z` | **z**ip（代指gzip）   | 使用gzip压缩/解压  | gzip是GNU zip的缩写，z为zip首字母              |
| `-j` | bz**j**2（代指bzip2） | 使用bzip2压缩/解压 | bzip2无直接首字母缩写，j是约定俗成的简写（早期bzip工具曾用-j） |
| `-J` | **J**xz（代指xz）     | 使用xz压缩/解压    | xz工具的官方约定，大写J区分小写j（bzip2）             |

通用功能类  

| 选项 | 英文全称                | 中文释义                 | 补充说明                     |
|------|-------------------------|--------------------------|------------------------------|
| `-v` | **v**erbose             | 详细的/冗余的            | 显示操作的详细过程（文件列表、进度等） |
| `-f` | **f**ile                | 文件                     | 必须指定归档文件路径，`-f` 后紧跟文件名（如 `-f test.tar.gz`） |

> `-f` 后紧跟文件名

补充说明

1. `tar` 本身是 **t**ape **ar**chive（磁带归档）的缩写，早期用于磁带备份，因此选项设计偏向简洁（单字母）；
2. 压缩类选项（`-z/-j/-J`）是“归档+压缩”的组合：`tar -czf` = 创建+gzip压缩+指定文件，`tar -xjf` = 解压+bzip2解压+指定文件；
3. `-f` 是唯一必须跟参数的选项，且惯例放在所有选项最后（如 `tar -czvf test.tar.gz dir/`，而非 `tar -f test.tar.gz -czv dir/`）。

举例说明

打包

``` shell
tar -cjvf tmplog.tar.bz2 /tmp/log # 打包/tmp/log到当前文件夹，名字为tmplog.tar.bz2。注意：打包只会打包log文件夹下的内容，tmp下其他文件夹不会打包
tar -cjvf ~/tmplog.tar.bz2 /tmp/log # 打包/tmp/log到用户目录下，名字为tmplog.tar.bz2
```

解压缩

``` shell
tar -xzvf log.tar.gz       # 解压log.tar.gz到当前目录
tar -xzvf log.tar.gz -C ~/ # 解压log.tar.gz到用户目录
```

# ps  

``` bash
ps [options]
```

以下是几个常用 `ps` 命令选项及其解释：

1. **-A, -e**：列出系统中的所有进程。
2. **-a**：显示终端上的所有进程，包括其他用户的进程。
3. **-u**：显示进程的详细信息，包括用户、CPU 使用率等。
4. **-x**：列出与终端无关的进程。
5. **-l**：显示长格式的输出，包括更多的进程信息。
6. **-p PID**：根据进程 ID (PID) 显示指定的进程。
7. **-C COMMAND**：根据命令名 (COMMAND) 显示与该命令相关的进程。
8. **-N**：显示与指定的格式不匹配的进程。
9. **-T**：显示当前终端下的所有进程。

``` bash
ps -ef
```

- `e`: 表示 "everyone"（所有人），这个选项会显示系统上所有用户的进程，而不仅仅是当前用户的进程。
- `f`: 表示 "full format"（完整格式），这个选项会以完整的格式显示进程信息，包括父进程 ID、启动时间、CPU 占用时间等。

``` bash
$ps -ef
UID  PID PPID C STIME  TTY  TIME      CMD
root  1   0   0 02:05  ?    00:00:01  init
```

UID：用户ID，表示运行该进程的用户。这里是root用户。  
PID：进程ID，是进程的唯一标识符。这里的PID是1。  
PPID：父进程ID，表示创建这个子进程的父进程编号。这里的PPID是0。  
C：CPU使用率，表示进程占用CPU的百分比。这里的C是0。  
STIME：进程启动时间，表示进程开始运行的时间。这里的STIME是02:05。  
TTY：终端类型，表示进程所在的终端。这里的TTY是?，表示该进程没有与终端关联。  
TIME：CPU时间，表示进程占用的CPU时间。这里的TIME是00:00:01，表示进程占用的CPU时间为1秒。  
CMD：命令名称，表示启动进程的命令。这里的CMD是init，表示这个进程是用来启动操作系统的。

``` bash

```

grep -n "pattern" file.txt

- "pattern" 是模式
- file.txt 是文件

过滤2种关键字


显示行号

# free

free 是 Linux 下最常用的内存状态查看命令，用于快速显示物理内存 (RAM) 和 交换分区 (Swap) 的总容量、已用、空闲及缓存使用情况，单位是 KB

```bash
/ # free
              total        used        free      shared  buff/cache   available
Mem:         482644      177520      124508         276      180616      293848
Swap:        119804           0      119804
```

---

物理内存（Mem）

``` bash
Mem:      482644    177520    124508       276    180616    293848
```

- **total：482644 KB**
  总物理内存 ≈ **471 MB**

- **used：177520 KB**
  系统进程、应用实际正在使用的内存 ≈ **173 MB**。

- **free：124508 KB**
  完全空闲、没被任何程序占用的内存 ≈ **121 MB**。

- **shared：276 KB**
  进程间共享内存

- **buff/cache：180616 KB**
  磁盘读写缓冲、文件缓存 ≈ **176 MB**。
  这部分**不是被程序独占**，内存紧张时系统会自动释放。

- **available：293848 KB**
  系统**真正还能分配给新程序**的内存 ≈ **286 MB**。
这是判断内存是否够用的**核心指标**。

交换分区（Swap）

```bash
Swap:     119804         0    119804
```

- 总交换空间 ≈ **117 MB**
- **used = 0**：**完全没有使用交换分区**

---

没有触发 Swap，说明**系统内存压力很小，运行流畅**

> Swap = 系统的 “内存备胎”。当物理内存不够用时，Linux 会把一部分暂时不用的数据，挪到硬盘 / Flash 上划出的一块空间里，这块空间就叫 Swap 交换分区。

重点：**一旦开始用 Swap**说明真实内存已经不够了，系统被迫把数据挪来挪去，这就是内存压力大

# procrank

procrank 是嵌入式 Linux 系统中用于查看进程内存占用的命令行工具，按 PSS 从大到小排序，输出各进程的四类内存指标，常用于内存优化与泄漏排查。

| 指标 | 全称 | 核心含义 | 关键特点 |
| :--- | :--- | :--- | :--- |
| **VSS** | Virtual Set Size | 虚拟内存总量 | 含共享库、堆、栈等所有虚拟地址空间，**不反映实际物理内存** |
| **RSS** | Resident Set Size | 物理内存占用 | 实际驻留 RAM 的内存，**包含共享库**，会重复统计共享内存 |
| **PSS** | Proportional Set Size | 比例物理内存 | **按比例分摊共享库**，是更准确的系统内存占用指标（所有进程 PSS 之和≈系统物理内存） |
| **USS** | Unique Set Size | 独占物理内存 | 进程**完全独占**的内存，无共享，是定位进程自身内存开销的核心指标 |

# top

top查看cpu负载

```bash
Mem: 381684K used, 100960K free, 276K shrd, 1776K buff, 178272K cached
CPU: 27.6% usr  4.7% sys  0.0% nic 66.7% idle  0.0% io  0.0% irq  0.8% sirq
Load average: 1.33 1.29 0.91 7/365 12804
  PID  PPID USER     STAT   VSZ %VSZ CPU %CPU COMMAND
 1847  1845 root     S     822m174.1   2 21.3 speech_th /etc/conf/speech_th.conf
```

---

## 第一行：系统概况（通常显示 uptime 等信息，上述输出中省略了该行）

一般 `top` 的第一行会显示当前时间、系统已运行时间、登录用户数以及负载平均值。你给出的输出里没有这一行，但第二行直接是 `Mem`。

---

## 第二行：内存状态

```bash
Mem: 381684K used, 100960K free, 276K shrd, 1776K buff, 178272K cached
```

- **used**：已使用的物理内存（381684 KB ≈ 373 MB）
- **free**：完全未使用的物理内存（100960 KB ≈ 99 MB）
- **shrd**：被共享内存占用的量（276 KB，通常很小）
- **buff**：内核缓冲区占用的内存（1776 KB ≈ 1.7 MB）
- **cached**：页面缓存占用的内存（178272 KB ≈ 174 MB）

> **解读**：系统总物理内存 ≈ used + free = 482644 KB（约 471 MB）。虽然 `free` 看起来有 99 MB，但实际可用的内存还应包括 `buff` 和 `cached`（它们可以在需要时被快速回收）。所以可用内存 ≈ free + buff + cached ≈ 280 MB。内存使用率约为 79%，不算很高。

---

## 第三行：CPU 状态

```bash
CPU: 27.6% usr  4.7% sys  0.0% nic 66.7% idle  0.0% io  0.0% irq  0.8% sirq
```

- **usr**：用户空间进程占用 CPU 百分比（27.6%）
- **sys**：内核空间占用 CPU 百分比（4.7%）
- **nic**：低优先级（nice 值较高）用户进程占用（0.0%）
- **idle**：空闲 CPU 百分比（66.7%）
- **io**：等待 I/O 完成的时间（0.0%，说明没有磁盘/网络 I/O 瓶颈）
- **irq**：硬件中断占用（0.0%）
- **sirq**：软件中断占用（0.8%）

> **解读**：总 CPU 使用率 ≈ 33.3%（100% - 66.7%），其中用户态占了大头（27.6%），系统态只有 4.7%，说明主要负载来自普通进程而非内核操作。没有 I/O 等待，磁盘不慢。

---

## 第四行：负载平均值（Load Average）

``` bash
Load average: 1.33 1.29 0.91 7/365 12804
```

- **1.33**（1分钟平均负载）
- **1.29**（5分钟平均负载）
- **0.91**（15分钟平均负载）
- **7/365**：7 个可运行（正在运行或可运行）的进程，总共有 365 个进程（包括睡眠、僵尸等）
- **12804**：最近一次运行的进程 PID（或 `top` 运行以来的总进程数变化，不同版本含义略有差异）

> **解读**：负载平均值为 1.33，而你的系统是单核（根据后面 CPU 只有一个 CPU 编号 `CPU:2` 但实际可能是单核或虚拟核）。对于单核系统，负载 >1 意味着 CPU 略微过载（有进程在排队）。不过 1.33 只是轻度超载，且 15 分钟负载 0.91 表明近期负载在上升。整体压力不大，但需要关注。

---

## 第五行及之后：进程列表

```bash
  PID  PPID USER     STAT   VSZ %VSZ CPU %CPU COMMAND
 1847  1845 root     S     822m174.1   2 21.3 speech_th /etc/conf/speech_th.conf
```

- **PID**：进程 ID（1847）
- **PPID**：父进程 ID（1845）
- **USER**：运行用户（root）
- **STAT**：进程状态（`S` = 可中断睡眠，通常是等待事件）
- **VSZ**：虚拟内存大小（822 MB）
- **%VSZ**：VSZ 占总物理内存的百分比（174.1%，已超过物理内存大小，说明有大量虚拟内存，可能使用了 swap 或映射了大文件）
- **CPU**：该进程上次更新时所在的 CPU 核心编号（2）
- **%CPU**：CPU 使用率（21.3%，相当高）
- **COMMAND**：命令行（`speech_th /etc/conf/speech_th.conf`，看起来是一个语音相关或自定义的守护进程）

> **解读**：这个 `speech_th` 进程消耗了 21% 的 CPU 和超过物理内存的虚拟内存，是当前最主要的资源消耗者。状态 `S` 表示它大部分时间在睡眠（等待事件），但一旦唤醒就会占用大量 CPU。需要确认该进程是否正常。
